HTML

HTML (от англ. HyperText Markup Language — «язык гипертекстовой разметки») — стандартизированный язык разметки документов для просмотра веб-страниц в браузере. Веб-браузеры получают HTML документ от сервера по протоколам HTTP/HTTPS или открывают с локального диска, далее интерпретируют код в интерфейс, который будет отображаться на экране монитора.

Элементы HTML являются строительными блоками HTML страниц. С помощью HTML разные конструкции, изображения и другие объекты, такие как и нтерактивная веб-форма, могут быть встроены в отображаемую страницу. HTML предоставляет средства для создания заголовков, абзацев, списков, ссылок, цитат и других элементов. Элементы HTML выделяются тегами, записанными с использованием угловых скобок. Такие теги, как <img /> и <input />, напрямую вводят контент на страницу. Другие теги, такие как <p>, окружают и оформляют текст внутри себя и могут включать другие теги в качестве подэлементов. Браузеры не отображают HTML-теги, но используют их для интерпретации содержимого страницы.

Язык XHTML является более строгим вариантом HTML, он следует синтаксису XML и является приложением языка XML в области разметки гипертекста.

В HTML можно встроить программый код на языке программирования JavaScript, для управления поведением и содержанием веб-страниц. Также включение CSS в HTML описывает внешний вид и макет страницы.

Что такое DOCTYPE и зачем он нужен

Элемент <!doctype> — DTD (document type definition, описание типа документа) указывает тип документа, который используется при написании html кода. Это непарный тег, который должен стоять самым первым в коде каждой страницы вашего сайта.

Директива doctype необходима, чтобы браузер понимал как обрабатывать текущую веб-страничку, так как существует несколько версий языка HTML, а еще имеется XHTML (EXtensible HyperText Markup Language, расширенный язык разметки гипертекста), который похож на HTML, но различается с ним по синтаксису.

Поэтому если вы хотите, чтобы ваш сайт отображался одинаково красиво и ровно во всех популярных браузерах, то обязательно указывайте эту директиву с версией языка html, которую вы используете. Причем doctype следует указывать с общепринятыми правилами. Если вы будете пользоваться неполным тегом DOCTYPE, устаревшим его видом, или вообще забудете про него, то браузер перейдет в хитрый режим уловок - quirks mode. И возможно странички вашего сайта отобразятся правильно в каких-то браузерах, но не факт, что они будут выглядеть также аккуратно в других популярных браузерах.

Подведем итоги:

  • DOCTYPE — это элемент, который указывает тип документа и определяет правила по, которым браузер будет обрабатывать ваш html-код;
  • элемент doctype всегда должен быть размещен в первой строчке документа;
  • всегда указывайте doctype, иначе ваши странички могут отображаться некорректно;
  • если не знаете какой doctype использовать, указывайте doctype html5: <!doctype html>.

Версии

  1. HTML 1.0;
    • Официальной спецификации HTML 1.0 не существует. До 1995 года существовало множество неофициальных стандартов HTML. Чтобы стандартная версия отличалась от них, ей сразу присвоили второй номер.
    • Базовая версия HTML поддерживает основные элементы, такие как текстовые драйверы и изображения. Это была базовая версия HTML, которая не поддерживала несколько элементов HTML. У него нет богатых функций, таких как форматирование и другие вещи, связанные с созданием контента в браузере.
    • Исходная версия HTML не поддерживает таблицы, поддержку шрифтов и т. д., поскольку она предоставляет нам последнюю версию.
    • Мы также хотим обсудить, что W3C не существовало до HTML 2.0, поэтому он не показывает подробностей о HTML 1.
  2. HTML 2.0 — опубликован IETF как RFC 1866 в статусе Proposed Standard (24 ноября 1995 года);
    • HTML версии 2.0 был разработан в 1995 году с целью улучшения HTML версии 1.0.
    • Теперь стандарт начал разрабатываться для поддержки единых правил и настроек во всех браузерах. HTML 2.0 был значительно улучшен с точки зрения тегов разметки. В HTML 2.0 вступило в силу понятие формы. Формы были разработаны, но все еще имели базовые теги, такие как текстовые поля, кнопки и т. д.
    • Таблица также появилась в виде тега HTML. Теперь, с HTML Tag 2.0, браузеры также придумали концепцию создания собственных слоев тегов, специфичных для самого браузера. Также был сформирован W3C. Основная цель W3C — поддерживать стандарт в разных браузерах, чтобы эти браузеры одинаково понимали и представляли HTML-теги.
  3. HTML 3.0 — 28 марта 1995 года — IETF Internet Draft (до 28 сентября 1995 года);
  4. HTML 3.2 — 14 января 1997 года;
    • В HTML версии 3.2 теги HTML были улучшены. Следует отметить, что из-за поддержки стандарта W3C более новая версия HTML была 3, 2 вместо 3.
    • HTML 3.2 теперь лучше поддерживает новые элементы формы. Еще одной важной функцией, реализованной в HTML 3.2, была поддержка CSS. CSS означает таблицу стилей CSS. CSS предоставляет функции, которые улучшают внешний вид тегов HTML при отображении в браузерах. CSS помогает стилизовать HTML-элементы.
    • Когда браузеры были обновлены до HTML 3.2, браузер также поддерживал теги фреймов, хотя конфигурации HTML по-прежнему не поддерживают теги разметки фреймов.
  5. HTML 4.0 — 18 декабря 1997 года;
  6. HTML 4.01 — 24 декабря 1999 года;
    • Он был разработан в 1999 году и расширил поддержку листов форматирования CSS. В версии 3.2 CSS был встроен в саму HTML-страницу. Поэтому, если на веб-сайте есть несколько веб-страниц, к которым можно применить стиль каждой страницы, нам необходимо разместить CSS на каждой веб-странице. Поэтому повторялся один и тот же блок CSS.
    • Чтобы решить эту проблему, в версии 4.01 родилась концепция внешнего листа проектирования. В соответствии с этой концепцией может быть разработан внешний файл CSS, и этот внешний файл форматирования может быть включен в сам файл HTML. HTML 4.01 обеспечивает поддержку новых тегов HTML.
  7. SO/IEC 15445:2000 (так называемый ISO HTML, основан на HTML 4.01 Strict) — 15 мая 2000 года;
  8. HTML5 — 28 октября 2014 года[13];
    • Это последняя версия HTML. Для разработчика он может быть использован в 2014 году. В нем было много HTML-тегов. HTML5 поддерживает новые элементы формы, такие как различные типы элементов ввода, теги поддержки данных о местоположении и т. д.
  9. HTML 5.1 начал разрабатываться 17 декабря 2012 года. Рекомендован к применению с 1 ноября 2016 года.
  10. HTML 5.2 был представлен 14 декабря 2017 года.
  11. HTML 5.3 был представлен 24 декабря 2018 года.

Давайте посмотрим на несколько тегов, которые были добавлены в HTML5.

  • Электронная почта. Новый тег HTML5, который был добавлен, является элементом ввода для типа электронной почты. Это тег формы, хотя его можно использовать вне тега формы. Этот тег проверяет правильность входного значения. Он проверяет, является ли введенное значение действительным адресом электронной почты.
  • Пароль — это еще один тег формы, который был добавлен для получения пароля. Потому что поле типа пароля, которое пользователь вводит в поле, не отображается непосредственно пользователю, а обозначается специальными символами. Эти символы предотвращают раскрытие пароля в браузере.
  • Голосовой тег — это новый голосовой тег, представленный в HTML5. Этот тег поможет увеличить объем нашего сайта. Мы можем использовать этот тег для встраивания аудиоклипа на веб-страницу. Эту голосовую метку можно воспроизвести на веб-странице.

  • Тег Cut — этот тег используется для семантической обработки части HTML-страницы. Тег section представляет собой часть веб-страницы.
